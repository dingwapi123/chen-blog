# Chen Blog 视觉设计规范

## 设计主张

**安静的编辑感。** 通过克制的暖灰背景、深青绿强调色、清晰的文字层级和细微的抽象纹理，让博客像一份长期维护的个人刊物，而不是通用内容模板。

首页的任务是建立作者与内容的第一印象；列表页强调扫读；文章页优先保证长时间阅读；CMS 则以高信息密度和明确操作为先。界面不使用营销式卡片墙、重阴影、第二强调色或外部摄影素材。

## 主题与 Token

所有颜色必须通过语义 token 使用，组件中不得直接写十六进制颜色。浅色与深色共用 token 名称。

| Token | Light | Dark | 用途 |
| --- | --- | --- | --- |
| `--bg` | `#F7F5F0` | `#171B1E` | 页面背景 |
| `--bg-soft` | `#EEECE6` | `#1E2528` | 柔和区域与纹理 |
| `--bg-elevated` | `#FFFDF9` | `#252D30` | 导航、弹层与代码表层 |
| `--text` | `#1F1F1D` | `#ECE9E2` | 主文字 |
| `--text-muted` | `#61615D` | `#B1AEA8` | 辅助文字 |
| `--text-faint` | `#8B8881` | `#7D8586` | 弱提示 |
| `--border` | `#DDD6CA` | `#34393E` | 常规边界 |
| `--border-strong` | `#C9C1B4` | `#4A5356` | 交互边界 |
| `--accent` | `#0F8A5F` | `#45C597` | 唯一强调色 |
| `--accent-hover` | `#0C7752` | `#58D5AA` | 强调色 hover |
| `--accent-soft` | `#D8F2E7` | `#103B2E` | 标签与轻提示 |
| `--code-bg` | `#EBE8E1` | `#101517` | 代码背景 |

主题默认跟随系统。用户点击切换后存储明确的 light/dark 选择；前台通过 SSR cookie 与首屏脚本应用主题，CMS 在 `index.html` 的首屏脚本中应用主题。所有过渡在 `prefers-reduced-motion: reduce` 下关闭。

## 字体、节奏与布局

- UI、导航、按钮与标题：`Avenir Next`, `PingFang SC`, `Microsoft YaHei`, sans-serif。
- 文章正文：`Source Han Serif SC`, `Noto Serif SC`, `Songti SC`, serif。
- 代码：`JetBrains Mono`, `Fira Code`, ui-monospace, monospace。
- 全局基准字号为 `16px`，文章行高 `1.9`；首页正文与管理界面行高 `1.78`。
- 间距 token：`2xs 0.35rem`、`xs 0.55rem`、`sm 0.8rem`、`md 1.15rem`、`lg 1.8rem`、`xl 2.8rem`、`2xl clamp(3.2rem, 7vw, 5.2rem)`、`3xl clamp(4.2rem, 9vw, 7rem)`。
- 公开页容器最大宽度为 72rem；文章正文为 44rem；移动端保留至少 1.25rem 内容边距。

## 页面构图

### 公开前台

首页首屏使用全宽大标题、站点名与一块低对比度 CSS 网格/弧线纹理；文案区域保持窄而明确，行动入口仅为“阅读文章”和“关于我”。以下依次是最新文章、关注主题和简短作者介绍。文章列表采用带分隔线的行式布局，标题优先于图片。文章页使用窄阅读列、清晰元信息、Markdown 内容、标签与上一篇/下一篇导航。

页头保持轻薄，滚动后仅提升背景层级；页脚只展示版权、简短说明与可替换社交链接。导航、标签和按钮圆角适中，不使用胶囊堆叠。

### CMS

CMS 为单管理员工作台：桌面端左侧导航、主工作区、顶部快捷操作；移动端切换为抽屉导航。文章列表、编辑器、分类、标签和媒体各有单一操作焦点。状态通过文字、边框与唯一强调色表达，不用多色仪表盘。

## 交互与动效

1. 首页首屏文字与纹理以 180–260ms 的淡入上移出现。
2. 文章列表标题 hover 改变强调色，并让箭头或下划线轻微位移。
3. 主题切换仅过渡颜色与背景，持续不超过 180ms。

动画不改变信息结构、不自动循环，且键盘焦点始终清晰可见。文章代码块必须可横向滚动；所有可点击图标有文本标签或无障碍名称。

## 可替换内容

V1 使用“陈信至 / 技术博客”为默认站点身份，并提供示例简介、文章、标签与社交链接占位。它们集中在前台站点配置与本地 Supabase seed 中，后续替换不应改动组件或数据库结构。
